PRを出す前のセルフレビューです。いまの差分を「別人のレビュアー」として批評し、指摘を直してください。

1. `git diff` と、関連仕様 `specs/<NNNN>/spec.md`・`AGENTS.md`・（あれば）`REVIEW.md` を読む。
2. 差分だけを新しい視点で評価:
   - 正しさ（ロジック / エッジケース / 競合 / 例外処理 / 後方互換）
   - セキュリティ（入力検証 / 認可 / PII・シークレット混入 / インジェクション）
   - 仕様適合（受け入れ条件を満たすか、ルール違反はないか）
   - テスト（受け入れ条件に対応するテストがあるか）
3. 指摘を重大度（Important / Nit）付きで列挙。**Important は必ず修正**、Nit は任意。
4. 修正後、テスト/リント/ビルドを実行して緑を確認し、結果（証拠）を示す。

正しさ・仕様/ルール違反に集中し、無関係なスタイル指摘で過剰設計しないこと。
