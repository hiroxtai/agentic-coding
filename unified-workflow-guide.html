01

結論と全体像

3つの手法を1つに束ね、3つのエージェントで同じ流れを回す。鍵は「共有コア＋アダプタ」。

このガイドは、これまで個別に見てきた 仕様駆動開発・インタビュー深掘り・ADR引き継ぎ を 1つのワークフローに統合し、GitHub Copilot / Claude Code / OpenAI Codex のどれでも、 どのプロジェクト・どのタスクでも同じ手順で開発できる構成を、具体的な手順としてまとめます。

◆ 設計の核（1文）
AGENTS.md を全エージェント共通の“背骨”にし、Spec Kit はエージェント別に導入、 grill-me は移植可能な Agent Skill として共有、ADR＋memory-bank で引き継ぐ。 仕様・決定・進捗はツール非依存のファイルとして1か所に置くので、 エージェントを切り替えても文脈が途切れません。
▶ 付随する構築キットと対で使う
本手順書は、別途用意した実ファイル一式 spec-driven-handoff-kit/（AGENTS.md・ constitution・skills・prompts・memory-bank・specs 見本）と対です。 ここで全体像と手順を理解し、キットのファイルを自分のリポジトリにコピーして構築します。
02

アーキテクチャ：共有コア＋アダプタ

1つの共有コアを、3エージェントが各自のアダプタ越しに読み書きする。これが横断の要。

✔ なぜ横断できるのか
AGENTS.md は3エージェントすべてが読む共通標準。Spec Kit は同じ specs/ を生成・更新し、grill-me は移植可能な SKILL.md 形式、 ADR/progress は素のMarkdown。共有コアが素のファイルだからこそ、ツールを選ばない。
03

3本柱の役割分担

統合といっても、3つの手法は競合しない。役割が違い、ワークフローの別々の場面で効く。

柱役割（いつ効くか）道具成果物
① インタビュー深掘り実装前に要件・制約・前提・代替案を引き出し、曖昧さを潰すgrill-me（Skill）／ Claude の AskUserQuestion固まった意図 → spec の下書き
② 仕様駆動意図をspec→plan→tasks→実装へ段階的に降ろし、推測を消すSpec Kit（/speckit.*）＋ constitutionspecs/<NNNN>/ の3点セット
③ ADR引き継ぎ決定と進捗を窓の外に残し、次セッション/別エージェントへ継承memory-bank（ADR・progress）＋各ネイティブ記憶ADR・progress・architecture
◆ 3本柱がつながる瞬間
grill で固めた意図が spec になり（①→②）、spec から実装が生まれ、その過程の決定が ADR として残り（②→③）、次のセッションは ADR と progress を読んで再開する（③→①）。 この円環が次章のワークフローです。
04

8ステップ・ワークフロー

どのストーリー/タスクでも、毎回この円環を回す。色は3本柱の対応を表す。

05

インタビュー深掘り（grill-me）

「容赦なく質問して意図を引き出す」スキル。曖昧なまま実装に入る事故を防ぐ、ワークフローの入口。

grill-me（Matt Pocock 発の Agent Skill）は、計画や仕様を質問攻めで圧迫テストし、 意図・制約・隠れた前提・語られていない代替案を引き出すスキルです。Claude Code・Codex・Copilot ほか40以上のエージェントが使う SKILL.md 形式で、本キットにも汎用版を同梱しています。 Claude Code では標準の AskUserQuestion でも同様の自動インタビューができます。

起動の合図は 「grill me on…」「interview me about…」「pressure-test this…」 など。 本キットでは Copilot は /grill-me（prompt file）、Claude Code / Codex は grill-me スキルで呼びます。結論は specs/<NNNN>/spec.md に反映し、 重要な判断は ADR 追記を促します。

06

仕様駆動（Spec Kit）

深掘りで固めた意図を、憲法→spec→plan→tasks→実装へ段階的に降ろす。エージェント別に導入する。

grill で固めた意図を、Spec Kit の4フェーズで実装へ落とします。土台に 憲法（constitution）を置き、各フェーズはチェックポイントで人がレビューします。 Spec Kit は3エージェントすべてに対応し、使うエージェントごとに導入します。

# プロジェクトに Spec Kit を導入（使うエージェントごとに1回）
specify init . --integration copilot
specify init . --integration claude
specify init . --integration codex
# 版により --ai の場合あり。非対話時の既定は copilot。`specify init --help` で確認

# 以降、各エージェントのチャットで（Copilotは prompt file、Claude/Codex は skill として）
/speckit.constitution   # 不変原則を確立 → .specify/memory/constitution.md
/speckit.specify        # 何を・なぜ → specs/<NNNN>/spec.md
/speckit.plan           # スタック・構成・制約 → plan.md
/speckit.tasks          # 検証可能な小タスク → tasks.md
/speckit.implement      # タスクを1つずつ実装
▶ 詳細は専用資料へ
Spec Kit の思想・4フェーズ・コマンドの詳細は、別資料「GitHub Spec Kit 詳解」にまとめています。 本ワークフローでは、生成される specs/<NNNN>/ が引き継ぎ資産も兼ねる （次セッションが読む）点が重要です。Spec Kit を使わず、AGENTS.md の指示だけで素の Markdown として spec/plan/tasks を書く運用でも成立します。
07

ADR引き継ぎ（memory-bank）

決定と進捗を窓の外のファイルに残し、次セッション・別エージェントへ確実に継承する。

セッションをまたぐ記憶は、会話ではなく memory-bank/ のファイルに置きます。 各エージェントのネイティブ記憶（Copilot Memory・Codex memories・Claude のメモリ）は補助で、 自動・確率的・失効ありのため、確定情報は必ずファイル側に持ちます。

  • progress.md：現在地・次の一手・最近の変更。開始時に読み、終了時に更新（心臓部）。
  • architecture.md：変わりにくい構造の地図。大きな変更時だけ更新。
  • decisions/（ADR）：重要な判断を1決定1ファイルで連番記録。追記のみ。 変更は過去ADRを消さず supersede する（「進化 vs 上書き」問題を回避）。
✔ 別エージェントへの引き継ぎも同じ仕組み
相手のエージェントも AGENTS.md と memory-bank/ を読むので、 Copilot で詰めた spec を Claude Code が実装し、決定は Codex も読める。 1つの共有コアが、ツール間の引き継ぎをそのまま可能にします。詳細は別資料 「セッション間引き継ぎ 詳解」を参照。

09

構築手順（クイックスタート）

付随キット spec-driven-handoff-kit/ を使えば5段階で構築できる。小さく始めて足していく。

# 1) 共有コアをコピー
cp -r shared/AGENTS.md shared/memory-bank shared/specs <your-repo>/
mkdir -p <your-repo>/.specify/memory && cp shared/constitution.md <your-repo>/.specify/memory/

# 2) Spec Kit（使うエージェント分）
specify init . --integration copilot   # 必要に応じ claude / codex も

# 3) アダプタ配置
cp copilot/copilot-instructions.md <your-repo>/.github/copilot-instructions.md
cp copilot/prompts/*.prompt.md      <your-repo>/.github/prompts/
cp -r skills/*                      <your-repo>/.claude/skills/   # Codex は各自の skills へ

# 5) コミット
git add AGENTS.md .github .claude .specify memory-bank specs
git commit -m "Add unified spec-driven handoff workflow"
10

注意点・使い分け

統合は強力だが、入れすぎは逆効果。最小から育てる。

  • 小さく始める：まず AGENTS.md＋memory-bank/＋grill-me から。Spec Kit と3エージェント併用は後で足す。
  • 指示は短く保つ：AGENTS.md / constitution が肥大化すると重要指示が埋もれる。「消すと間違えるか？」がNoなら削る。
  • 確定情報はファイルに：ネイティブ記憶（Copilot Memory・Codex memories）は補助。失効・確率的なので頼り切らない。
  • ターンキー代替：自前構築の代わりに、クロスエージェントSDDハーネス cc-sdd（Agent Skills で Claude/Codex/Copilot 対応）を土台にする手も。「共有コア＋アダプタ」の考え方はそのまま使える。
  • 版差に注意：specify init --help、各エージェントのスキル/プロンプト配置は最新ドキュメントで確認する。
◆ ひとことで
「1つの共有コア（AGENTS.md＋specs＋memory-bank）を、3エージェントがアダプタ越しに読み書きする」 ―― これさえ守れば、どのプロジェクト・どのタスク・どのエージェントでも、深掘り→仕様→実装→引き継ぎの同じ流れで開発できます。
