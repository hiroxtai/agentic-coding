この資料について

「手順書」を読んだ後の実演編。実際の1機能を、最初から最後までエージェントと進める様子を見せる。

別資料「統合エージェント開発ワークフロー 構築手順書」で構成と手順を示しました。本資料はその実演編です。 「パスワードリセット機能の追加」という現実的な1タスクを、あなたが打つ指示と エージェントの動きのやり取りとして、Day 1〜Day 2 にわたって追います。

▶ 読み方
▷ あなた = あなたが打つ指示／プロンプト、 エージェント = その応答（簡略）、 あなた: … = grill への短い回答、です。

シナリオと前提

統合キット導入済みのプロジェクトに、新しいストーリーが入ってきた、という想定。

  • プロジェクト：Express + TypeScript の API（DB は Prisma + PostgreSQL）。
  • 導入済み：AGENTS.md / .github/prompts/ / memory-bank/ / specs/（統合キット）。Copilot Memory も ON。
  • 既存機能：ユーザー登録・ログインのみ（src/auth/）。メール送信基盤は未実装。
  • 今回のタスク：「パスワードリセット機能を追加したい」。

Step 5｜実装

仕様駆動
▷ あなたtasks.md の 1番から順に実装して。各タスクごとにテストを書いて npm test を通してから次へ。1タスク終わるたびに止めて差分を見せて。

進め方の要: 気づいた時点でその場で短く指示すればよい（長いプロンプトを練り直さない）。今日はタスク4まで進んで時間切れ、とします。

10

早見表と3つの核心

局面ごとに打つ“合言葉”はこれだけ。覚えるのは3つの原則。

局面打つ指示
開始時/resume-session
要件が曖昧/grill-me <やりたいこと>
計画を固めたい/speckit.plan <スタック・制約> → 「この plan を grill して」
実装「tasks の N番から、1つずつテストを通して、終わるごとに差分を見せて」
決定が出た「この決定を ADR に残して」
中断・終了時/update-handoff → commit
◆ 覚えるのは3つだけ
① 開始＝/resume-session、終了＝/update-handoff を必ず挟む。 ② 曖昧なら実装前に /grill-me。 ③ 実装は1タスクずつ＋検証＋差分確認。 これさえ守れば、どのプロジェクト・どのタスク・どのエージェントでも同じ型で進められます。
