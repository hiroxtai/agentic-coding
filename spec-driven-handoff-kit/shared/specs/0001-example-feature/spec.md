## 背景・目的（Why）
ユーザーがパスワードを忘れた場合に、自力で安全に再設定できるようにする。
（解く問題: サポート問い合わせの削減とログイン継続率の向上）

## 利用者ストーリー（Who / Journeys）
- ログインできないユーザーが「パスワードを忘れた」を選び、メールアドレスを入力する。
- 受信したメールのリンクから、新しいパスワードを設定し、ログインできる。

## 受け入れ条件（Acceptance criteria）
- [ ] 未登録メールでも「送信しました」を表示する（アカウント存在を漏らさない）。
- [ ] リセットリンクは有効期限つきで、1回のみ使用可能。
- [ ] 新パスワードはポリシー（長さ等）を満たす場合のみ受理する。

## スコープ外（Out of scope）
- ソーシャルログインのパスワード（対象外）。

## 検証（E2E verification）
- ブラウザ操作で「忘れた→メール→リンク→新パスワード→ログイン成功」を一気通貫で確認する。
